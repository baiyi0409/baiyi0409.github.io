.Net对接微信支付

2023-11-06
.Net
11k words
  • .NET

简单介绍

本次使用官网提供的JSAPI接口，完成收款以及退款的流程

使用的库 SKIT.FlurlHttpClient.Wechat

支持 .NET Framework 4.6.1+、.NET Standard 2.0+、.NET Core 2.0+、.NET 5+ 等环境

主要使用的是 SKIT.FlurlHttpClient.Wechat.TenpayV3 中的功能 (微信支付 v3 版 API)

DotNetCore.SKIT.FlurlHttpClient.Wechat: 可能是全网最完整的 C# 版微信 SDK，封装全部已知的微信 API，包含微信公众平台（订阅号+服务号+小程序+小游戏+小商店+视频号）、微信开放平台、微信商户平台（微信支付+微企付）、企业微信、微信广告平台、微信智能对话开放平台等模块，可跨平台。持续随官方更新

JSAPI接口介绍

JSAPI支付是指商户通过调用微信支付提供的JSAPI接口，在支付场景中调起微信支付模块完成收款。

具体操作流程如下：

步骤一 如图1，商户下发图文消息或者通过自定义菜单吸引用户点击进入商户网页。

步骤二 如图2，进入商户网页，用户选择购买，完成选购流程。

步骤三 如图3，调起微信支付控件，用户开始输入支付密码。

步骤四 如图4，密码验证通过，支付成功。商户后台得到支付成功的通知。

5.调用JSAPI下单接口

微信官方文档：JSAPI下单 - JSAPI支付 | 微信支付商户文档中心 (qq.com)

这里只填入必填项，选填项请看官方文档根据需求进行选填

需要传入的参数

  • appid：【公众号ID】 公众号ID
  • mchid： 【直连商户号】 直连商户号
  • description ：【商品描述】 商品描述
  • out_trade_no：【商户订单号】 商户系统内部订单号，只能是数字、大小写字母_-*且在同一个商户号下唯一。
  • notify_url：【通知地址】 异步接收微信支付结果通知的回调地址，通知URL必须为外网可访问的URL，不能携带参数。 公网域名必须为HTTPS，如果是走专线接入，使用专线NAT IP或者私有回调域名可使用HTTP 这里传的是功能设置里面的网页授权域名
  • amount：【订单金额】 订单金额信息
    • total：【总金额】 订单总金额，单位为分
  • payer：【支付者】 支付者信息
    • openid：【用户标识】 用户在普通商户AppID下的唯一标识

返回参数

  • prepay_id：【预支付交易会话标识】 预支付交易会话标识。用于后续接口调用中使用，该值有效期为2小时

notify_url 设置位置

其中获取openid请查看官方文档：微信网页开发 / 网页授权 (qq.com)

openid获取流程

第一步：用户同意授权，获取code

引导用户进行授权，相当于点击按钮访问接口地址

https://open.weixin.qq.com/connect/oauth2/authorize?appid=APPID&redirect_uri=REDIRECT_URI&response_type=code&scope=SCOPE&state=STATE#wechat_redirect

  • scope：应用授权作用域，snsapi_base （不弹出授权页面，直接跳转，只能获取用户openid），snsapi_userinfo （弹出授权页面，可通过openid拿到昵称、性别、所在地。并且， 即使在未关注的情况下，只要用户授权，也能获取其信息 ）

如果用户同意授权，页面将跳转至 redirect_uri/?code=CODE&state=STATE，这里code，用来下一步请求access_token使用

第二步：通过code换取网页授权access_token

获取code后，请求以下链接获取access_token：

https://api.weixin.qq.com/sns/oauth2/access_token?appid=APPID&secret=SECRET&code=CODE&grant_type=authorization_code

返回参数

这时候就获取到了openid

第三步：刷新access_token（不必要）

此接口主要用于刷新token(当token失效后，需要用户重新授权)

第四步：拉取用户信息(需scope为 snsapi_userinfo)(不必要)

此接口主要用于获取用户信息，大致为昵称、性别、头像、省份等

6.JSAPI调起支付

通过JSAPI下单接口获取到发起支付的必要参数prepay_id，然后使用微信支付提供的前端JS方法调起公众号支付

这个步骤主要是将下单获取到的参数，以及生成签名传给前端，前端进行公众号支付

请求参数

  • appid：【公众号ID】 公众号ID
  • timeStamp：【时间戳】 标准北京时间，时区为东八区，自1970年1月1日 0点0分0秒以来的秒数。注意：部分系统取到的值为毫秒级，需要转换成秒(10位数字)
  • nonceStr：【随机字符串】 不长于32位
  • package：【prepay_id】 ** JSAPI下单接口返回的prepay_id参数值，提交格式如：prepay_id=*
  • signType：【签名类型】 默认为RSA，仅支持RSA
  • paySign：【签名】 使用字段AppID、timeStamp、nonceStr、package计算得出的签名值

计算签名值使用的是 SHA256 with RSA签名，并对签名结果进行 Base64编码 得到签名值

7.调用退款接口

微信官方文档：退款申请 - JSAPI支付 | 微信支付商户文档中心 (qq.com)

HTTP 头信息不变，同样这里也只传入必传的参数，有需要请看文档添加选填参数

请求参数

  • transaction_id:【微信支付订单号】 原支付交易对应的微信订单号，与out_trade_no二选一
  • out_trade_no:【商户订单号】 原支付交易对应的商户订单号，与transaction_id二选一

上面两个参数二选一，是必填其中一个

  • out_refund_no:【商户退款单号】 商户系统内部的退款单号，商户系统内部唯一，只能是数字、大小写字母_-|*@ ，同一退款单号多次请求只退一笔。
  • amount:【金额信息】 订单金额信息
    • refund:【退款金额】 退款金额，单位为分，只能为整数，不能超过原订单支付金额
    • total:【原订单金额】 原支付交易的订单总金额，单位为分，只能为整数
    • currency:【退款币种】 符合ISO 4217标准的三位字母代码，目前只支持人民币：CNY
